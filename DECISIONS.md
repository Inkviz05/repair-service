# DECISIONS

1. Выбран Django + server-rendered templates вместо SPA, чтобы быстро и надежно закрыть ТЗ с минимальным инфраструктурным кодом.
2. Использована SQLite для простого локального старта и совместимости с `docker compose up` без отдельного DB-сервиса.
3. Роли реализованы через встроенные `Group` (`dispatcher`, `master`) и проверки в service/view-слое.
4. Переходы статусов централизованы в `services.py`, чтобы избежать дублирования логики в контроллерах.
5. Для защиты от гонки в действии `take` применен атомарный conditional `UPDATE ... WHERE status='assigned' ...`; при неуспехе возвращается `409 Conflict`.
6. Добавлен `RequestEvent` (audit log), фиксирующий ключевые действия и смену статусов.
7. Добавлены автотесты для критичных сценариев жизненного цикла заявки и проверки конфликтного `take`.
